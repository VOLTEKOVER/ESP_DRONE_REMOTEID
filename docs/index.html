1. Introduction

▼

What is ESP DRONE REMOTEID?

ESP DRONE REMOTEID is an open-source firmware being developed to transform any ESP32 microcontroller into a universal Remote ID transmitter targeting ASTM F3411-22a (USA) and ASD-STAN prEN 4709-002 (EU) standards. It is designed to read GPS position data from a flight controller via UART and broadcast ASTM-standard Open Drone ID messages over WiFi Beacon (802.11) and Bluetooth Low Energy (BLE 4.0 and BLE 5.0 Coded PHY).

⚠ Development Status: The firmware code is feature-complete for v1.0.0-beta. A built-in demo mode simulates a GPS patrol over Rome (Colosseum) so you can test the full config UI, OTA flow, and telemetry dashboard without flight controller hardware. Actual flight controller (FC) hardware verification with MAVLink/MSP/NMEA is ongoing. See the software status for the complete per-file analysis.

Implemented Features

  • Triple protocol input: MAVLink (ArduPilot), MSP (Betaflight/iNAV), NMEA 0183 parsers
  • Automatic protocol detection on boot — no manual switching needed
  • Dual radio broadcast: WiFi Beacon (802.11) + WiFi NAN + BLE 4.0 + BLE 5.0 Long Range Coded PHY
  • Built-in demo mode — GPS patrol simulation for testing without FC hardware
  • Built-in WiFi Access Point with web configuration at 192.168.4.1
  • Over-the-Air (OTA) updates with mandatory SHA-256 checksum validation
  • Security lock levels: level 0 (open), level 1 (signed commands), level 2 (eFuse permanent)
  • eFuse permanent lock: one-time irreversible level 2 burn with "RID!" magic
  • 26 configurable parameters matching ArduRemoteID feature set
  • Browser-based flashing via WebSerial — no toolchain needed
  • 6 ESP32 targets via CI: ESP32, ESP32-S2, ESP32-S3, ESP32-C2, ESP32-C3, ESP32-C6

Regulatory Compliance

The firmware targets the ASTM F3411-22a Open Drone ID (ODID) message set. The following message types are implemented in code:

  • Basic ID Message — UAS ID, ID type, UA type
  • Location/Vector Message — latitude, longitude, altitude, speed, heading
  • System Message — operator ID, timestamps
  • Self-ID Message — additional operator information

Note: Compliance with FAA Part 89, EASA 2021/664, or any other regulation has not been certified. Regulatory validation with certified receivers is required before use in actual flight operations.

Connect to any unused serial port (e.g. SERIAL4). Configure in Mission Planner:

SERIAL4_PROTOCOL = 1
SERIAL4_BAUD = 57
SERIAL4_OPTIONS = 0

Wire: FC TX → ESP32 GPIO16. GND → GND.

⚙️

Betaflight / INAV

set serialrx_provider = MSP
set serial_baudrate = 57600
Save & Reboot

Wire: FC TX3 → ESP32 GPIO16. GND → GND.

🌍
NMEA Tap
NMEA Clone Method

Tap directly into the GPS module's TX line with a 1 kΩ series resistor:

GPS TX → [1kΩ] → GPIO16
GPS TX → FC RX (unchanged)

Works with: BN-880, BN-220, M8N, M10, NEO-6M/8M.

Power Options

  • From FC BEC: Connect to the 5V pad on your flight controller's servo rail or BEC output. Most BECs provide 5V @ 2-3A, sufficient for the ESP32 (~150 mA).
  • From USB: Power via the ESP32's USB port during configuration and testing.
  • From battery: Use a 5V step-down regulator (e.g. Matek 5V bec) connected to the main battery.

The ESP32 draws approximately 150 mA with WiFi Beacon + BLE active. Ensure your BEC or regulator can supply this.

Antenna Selection

⚠ Important: Carbon fiber drone frames are conductive and will severely attenuate the ESP32's built-in PCB antenna. Use a board with a U.FL/IPEX connector for an external antenna.
  • PCB antenna: Suitable for glass fiber, plastic, or open frames. Range ~50-100 m.
  • External antenna (U.FL): Required for carbon fiber frames. Range ~200-500 m with 2 dBi antenna.
  • Recommended: ESP32-S3-DevKitC-1 with U.FL, or TTGO T-Internet-POE with external antenna.

Voltage Levels

The ESP32 GPIO pins are 5V tolerant but the UART RX pin (GPIO16) uses 3.3V logic levels. Most flight controllers use 3.3V UART (Pixhawk, Matek, Betaflight boards). If your FC uses 5V UART (some older ArduPilot boards), use a level shifter or a voltage divider (1 kΩ + 2.2 kΩ) on the RX line.

👤 Identity Stable

Basic ID — The UAS (Unmanned Aircraft System) ID identifies your drone to receivers in the area, as required by FAA/EASA/ENAC Remote ID regulations. Use a serial number following the ANSI/CTA-2063 format (4-char manufacturer code + length digit + serial) or your CAA registration number. Max 20 characters. The UA Type must match your aircraft category — "None" will be ignored by receivers. Wiki →

📡 Transmission Stable

Tx Modes — Select which radio technologies to broadcast the Remote ID message on. WiFi Beacon (IEEE 802.11) is the most compatible and recommended. BLE 4.0 is widely supported by smartphones. WiFi NAN and BLE 5.0 Coded PHY offer longer range but are experimental. Rates are in Hz (packets per second) — higher rates improve reception but increase airtime usage. Enable at least one mode for the ID to be visible. Wiki →
BLE 5.0 LR is experimental (ESP32-S3/C3). WiFi NAN experimental.

📶 Access Point Stable

Management Network — Configure the ESP32's built-in WiFi access point used for configuration. After connecting your phone or laptop to the SSID below, open http://192.168.4.1 in your browser. The password protects the config interface — leave empty for an open network. The AP runs on channel 6 by default, separate from the beacon transmission channel. Wiki →

🐳 Flight Controller Stable

Serial & Protocol — Configure the UART connection between the ESP32 and your flight controller (Pixhawk, ArduPilot, iNAV, Betaflight, etc.). MAVLink is used by ArduPilot/PX4, MSP by Betaflight/iNAV, NMEA for generic GPS modules. Set the baud rate to match your FC's serial port speed — 57600 or 115200 are common. Select Auto-detect to let the firmware probe all protocols at startup. Wiki →
Set to Auto unless detection fails. Manual override forces a specific protocol.

💡 Hardware Stable

Status LED — Configure GPIO pins for the RGB status LED (common cathode). Wire the red, green, and blue channels to separate GPIO pins through current-limiting resistors (typically 100–220Ω). Red = no GPS fix or error. Green = GPS 3D fix, normal operation. Blue = system booting or BLE active. Set unused channels to -1. Leave all at -1 to disable the LED entirely.
Set the GPIO numbers connected to your RGB LED cathode pins. Use -1 for unused channels.

🔒 System Warning Expert

Security & Options — Lock Level 0 (Normal) — no restrictions. Level 1 (Locked) — only accepts commands signed with a registered public key; the web UI remains accessible but factory reset via CLI requires authentication. Level 2 (eFuse permanent) — burns one-time programmable fuses to enforce lock; this cannot be undone and may brick the device if keys are lost! Set at least one public key before enabling lock levels 1 or 2. Wiki →
⚠ Level 2 burns eFuses permanently. This cannot be undone!

⚖ Compliance Checklist Beta

Regulatory compliance — Verify your current configuration against the requirements of your region's aviation authority. Select FAA (USA, Part 89), EASA (EU, Delegated Reg. 2019/945), CAA (UK), or ENAC (Italy). The checklist will highlight which mandatory fields are filled, missing, or incorrectly formatted. Use Auto-detect to infer the region from your current settings.

🔐 Security Beta

Panel access control — Optionally protect the web configuration panel with a password. When set, a login screen appears before any settings are shown. The password is stored in the device's NVS (non-volatile storage) and persists across reboots. To disable, clear the field and save. If you forget the password, a hardware reset via the button on the device will clear it. Session timeout controls how long the login lasts before re-authentication is required.
Min 4 chars. If set, login is required to access any page.
